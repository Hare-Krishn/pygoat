Lab Objective

This lab demonstrates how a high-risk security action can succeed without triggering any alerts. Your goal is to identify a privilege escalation that is logged but not detected.

  • Log in using the provided user account.
  • From the dashboard, request a role change using the available button.
  • Observe what happens when the request is denied. Does the role change? Does the system respond in any way?
  • Notice that the system appears to detect and react to this attempt.
  • Ask yourself: If alerts exist and are working, how could a similar action ever succeed silently?
  • Experiment with the same action while switching between Vulnerable and Secure modes. Pay close attention to what changes — and what does not.
  • Revisit the role change request and consider what the backend might be using to decide whether an alert should be triggered.
  • Continue experimenting until the same user is able to obtain administrative privileges without any warning or alert.

Success condition: the user alice is able to become an administrator without triggering any security alert.

Hint: Not all security failures come from missing logs.

If you get stuck or want to review the intended approach after completing the lab, a detailed walkthrough is available in the solution documentation for the Silent Privilege Escalation lab (see the project documentation or markdown files).
